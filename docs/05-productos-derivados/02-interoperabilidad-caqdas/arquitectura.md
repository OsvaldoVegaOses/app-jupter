# Arquitectura (Interoperabilidad CAQDAS)

## Objetivo
Garantizar que la evidencia y la estructura analítica (códigos/citas/memos/categorías) se exporten sin pérdida crítica.

## Componentes
- Exporter por formato (REFI-QDA, MAXQDA, NVivo, ATLAS.ti)
- Validador de paquete (esquema + consistencia)
- Dataset de pruebas (proyectos pequeños “golden”) para regresión

## Riesgos
- Ambigüedad semántica entre modelos de datos (APP_Jupter vs CAQDAS)
- Cambios de versiones y requisitos de importación
