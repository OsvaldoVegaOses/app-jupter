# Arquitectura (Enterprise Governance)

## Objetivo
Aportar gobernanza transversal: autenticación/SSO, auditoría, políticas de datos y operación.

## Componentes (alto nivel)
- Identity: SSO/OAuth + control de sesiones
- Audit logging: eventos de acceso, cambios de código, exportaciones, borrados
- Data governance: retención, backup, restauración, e-discovery (si aplica)

## Riesgos
- Requisitos varían por industria/jurisdicción
- El costo está en validación y operación, no solo en código
